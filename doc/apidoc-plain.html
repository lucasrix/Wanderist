Registrations / create

POST /auth
Email registration. A verification email will be sent to the email address provided.

Examples

POST /auth
{
  "email": "username@example.com",
  "password": "password",
  "password_confirmation": "password",
  "confirm_success_url": "http://localhost:3001/",
  "config_name": "default"
}
403
{
  "status": "error",
  "data": {
    "id": null,
    "provider": "email",
    "uid": "",
    "name": null,
    "nickname": null,
    "image": null,
    "email": "username@example.com",
    "created_at": null,
    "updated_at": null
  },
  "errors": {
    "email": [
      "already in use"
    ],
    "full_messages": [
      "Email already in use"
    ]
  }
}

Params

  • email : required [

    Must be String

    ]

    Email address

  • password : required [

    Must be String

    ]

    Password

  • password_confirmation : required [

    Must be String

    ]

    Password confirmation

Sessions / destroy

DELETE /auth/sign_out
Use this route to end the user's current session. This route will invalidate the user's authentication token.
